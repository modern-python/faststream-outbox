# Change: Split the schema-drift validator into schema_validation.py

**Lane:** lightweight in spirit — a pure behavior-preserving move with no
public-API change and no new test. It creates one new module (which normally
implies the Full lane), but there is no design judgment to record: the existing
100%-coverage suite is the spec, and every moved symbol is verbatim.

## Goal

Deepen the transport client by relocating the opt-in schema-drift validator from
`faststream_outbox/client.py` into a new sibling module
`faststream_outbox/schema_validation.py`. This concentrates the schema-drift
concern in one place, roughly halves `client.py` (796 → 471 LOC), and isolates the
optional `alembic` dependency (which backs `validate_schema()` only) in the module
that actually needs it.

## Approach

Pure move, zero behavior change. `OutboxClient.validate_schema` keeps its signature
and docstring but its body becomes a one-line delegation to a new module-level free
function `schema_validation.validate_schema(engine, table, *, dlq_table=None,
check_autovacuum=False)`. The new module is imported as a module
(`from faststream_outbox import schema_validation`) to avoid the method/free-function
name collision. No circular import: `schema_validation` imports only `schema`,
`autovacuum`, `_import_checker`, alembic, and sqlalchemy — never `client`.

Moved verbatim (comments intact): `validate_schema` body, `_normalize_predicate`,
`_validate_index_predicates_sync`, `_validate_check_constraints_sync`,
`_compose_schema_mismatch_message`, `_validate_schema_sync`,
`_validate_dlq_schema_sync`, `_run_validate`, `_flatten_drift_errors`,
`_drift_entry_to_error`, and their constants (`_EXPECTED_INDEX_PREDICATES`,
`_EXPECTED_UNIQUE_INDEXES`, `_INDEX_PREDICATE_QUERY`, `_CHECK_CONSTRAINT_QUERY`,
`_EXPECTED_CHECK_CONSTRAINTS`, `_SCHEMA_MISMATCH_PREFIX`, `_AUTOGEN_BLIND_HINT`).
The alembic guard block and the `autovacuum` / `MetaData` / `make_*` / index-suffix
imports move with them; the transport paths (`fetch`, `delete_*`, `mark_pending`,
`ping`, `_row_to_message`, `_build_dlq_cte_stmt`) and their imports stay.

Truth home: `architecture/schema.md` — the "message composition lives in
`_compose_schema_mismatch_message`" pointer moves from `client.py` to
`schema_validation.py` in this same diff.

## Files

- `faststream_outbox/schema_validation.py` — new module; the validator + free
  `validate_schema()`.
- `faststream_outbox/client.py` — validator removed; method delegates; unused
  imports pruned.
- `tests/test_unit.py` — five validator symbols repointed to `schema_validation`
  (`OutboxClient` still from `client`). No test body changed.
- `architecture/schema.md` — `client.py` → `schema_validation.py` pointer.
